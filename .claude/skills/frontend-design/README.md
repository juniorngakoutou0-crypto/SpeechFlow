# Frontend Design Skill

Create distinctive, production-grade frontend interfaces with exceptional design quality.

## What It Does

This skill helps you build beautiful, creative web interfaces that stand out from generic AI-generated designs. It focuses on:

- **Unique aesthetics** - Avoid overused patterns, fonts, and color schemes
- **Production quality** - Functional, accessible, performant code
- **Bold creativity** - Memorable designs with strong visual identity
- **Modern techniques** - Latest CSS features, animations, and effects

## When to Use

Invoke this skill when building:

- Websites and landing pages
- Web applications and dashboards
- React/Vue/Svelte components
- HTML/CSS layouts
- UI mockups and prototypes

## How to Use

### Basic Usage

Simply ask Claude to create a frontend component or page:

```
Build a landing page for a coffee shop
```

```
Create a pricing cards component in React
```

```
Design a dashboard for a fitness app
```

### Advanced Usage

For more control, specify:

- **Framework**: React, Vue, Svelte, or vanilla HTML/CSS
- **Aesthetic**: Brutalist, minimalist, retro-futuristic, editorial, etc.
- **Features**: Dark mode, animations, glassmorphism, etc.

Example:
```
Create a hero section in React with a cyberpunk aesthetic,
using neon colors and glassmorphism effects
```

## What Makes It Different

### ❌ Avoids Generic AI Patterns

- No Inter, Roboto, or Space Grotesk fonts
- No purple gradient on white backgrounds
- No predictable 3-column card layouts
- No cookie-cutter designs

### ✅ Delivers Distinctive Design

- Unique, contextual font choices
- Creative color palettes
- Unexpected layouts and compositions
- Modern CSS features (container queries, :has(), view transitions)
- Advanced visual effects (glassmorphism, gradient meshes, 3D transforms)
- Accessibility-first approach (WCAG 2.1 AA)
- Performance optimizations

## Tips for Best Results

1. **Be specific about the aesthetic** - "brutalist", "soft pastel", "retro-futuristic"
2. **Mention your audience** - "for developers", "luxury brand", "kids app"
3. **State technical requirements** - Framework, browser support, performance needs
4. **Request specific features** - Dark mode, animations, specific effects
